# Anaphase-Helix : 面向极低算力边缘集群的自主智能体迭代演化框架

**自主进化 | 资源约束适配 | 物理隔离架构 | 分层认知 | 商用模型兼容**

> 研究定位：低算力边缘环境下，基于资源最优原则的大模型智能体可持续演化系统。系统代号“洛无忌 (Helix Luo)”，以硅基生命体形态运行，在严格的 8K 上下文窗口与网关超时限制下，实现复杂研发任务的自主拆解、工具调用与认知进化。

---

## 摘要

Anaphase-Helix 是一套专为**私有内网ARM硬件集群**设计的自主智能体演化框架，聚焦 1GB RAM 低算力、高延迟边缘场景，构建了**生命周期迭代-工具化落地-经验固化-安全审计-最优筛选**的全闭环演化体系。框架摒弃传统大模型纯对话交互模式，以资源代谢守恒、工具化落地、帕累托最优为核心准则，实现智能体从被动任务执行向主动进化迭代的升级。系统支持本地私有模型（Tuck网关）与商用模型服务（OneAPI生态）无缝切换，通过物理级三层隔离架构保障运行安全，是面向边缘计算场景的轻量化自主智能体研究与落地平台。

## 1 项目概述

### 1.1 研究背景
传统大模型智能体框架（如OpenClaw）普遍依赖高算力硬件与公有云服务，在ARM架构、1GB RAM等低资源边缘环境中存在算力过载、延迟过高、资源浪费等问题；同时，多数框架仅支持被动任务执行，缺乏可持续的自我迭代与经验传承能力。

### 1.2 核心目标
本框架以**边缘低算力环境适配**为基础，以**智能体自主迭代演化**为核心，实现三大技术目标：
1. **工具化落地**：将重复业务逻辑封装为标准化Python工具，降低Token资源消耗；
2. **经验固化传承**：迭代周期内提炼有效认知，形成可复用的优化补丁，实现持续能力提升；
3. **资源最优平衡**：在计算资源消耗与智能体生产力之间建立帕累托最优机制。

### 1.3 核心优势
- 物理级三层隔离架构，杜绝单点故障与代码扩散风险；
- 原生适配ARM 1GB RAM低算力边缘节点；
- 模型层完全解耦，支持私有部署与商用服务无缝切换；
- 全流程安全审计，保障智能体执行合规性；
- 可回溯的迭代演化体系，支持智能体持续进化。

## 2 宏观架构（三位一体分布式拓扑）

系统严格部署于 `10.0.0.x` 私有内网，采用**认知-调度-执行**物理分离架构，满足工业级安全与稳定性要求：

| 节点IP | 模块定位 | 核心组件 | 核心职能 |
| :--- | :--- | :--- | :--- |
| **10.0.0.2** | 认知中枢 | Helix-Mind | 管理智能体生命周期、记忆存储、任务队列调度、人格服务 |
| **10.0.0.54** | 模型调度网关 | Tuck 人格路由引擎 | 多模型实例调度、人格隔离、请求路由，支持2B/4B/8B参数模型动态分配 |
| **10.0.0.206** | 安全执行沙箱 | Anaphase 内核 | 任务隔离执行、代码安全扫描、迭代成果校验、Git版本固化与回滚控制 |

> 架构特性：三层物理隔离设计，实现**认知管理、模型调度、安全执行**的解耦，兼顾演化效率与系统安全性。

## 3 智能体分工体系（三位一体流水线）

为突破 1GB 内存带来的物理计算瓶颈，系统摒弃了单一大模型方案，独创 **感知-认知-行动循环** 异构流水线：

| 角色 | 模型 | 核心职能 |
| :--- | :--- | :--- |
| **🧠 规划中枢 (Brain)** | DeepSeek-R1-8B | 接收极简系统提示，利用内生思维链 (CoT) 制定高维战术计划。 |
| **👁️ 认知蒸馏 (Eyes)** | Qwen-2B | 信息过滤器，对大模型的冗长计划和系统反馈进行强制“脱水”，将万字日志压缩为百字摘要。 |
| **🦾 物理执行 (Hands)** | Qwen-Coder-7B | 专精于代码生成的终端模型，接收脱水指令后，精确输出 `toolkit.xxx()` 工具调用代码。 |

**协作机制**：Brain 生成初步计划 → Eyes 压缩冗余信息 → Hands 执行物理动作 → 反馈循环迭代，直至任务完成。

## 4 核心技术特性

### 4.1 流式抗压与 KV 缓存接力 (Streaming & KV-Relay)
彻底解决 ARM 节点长文本预填充 (Prefilling) 导致的 504 网关超时。通过开启底层 Stream，击穿 HTTP 屏障，保障 8B 模型长序列思考的连续性。

### 4.2 动态认知分级 (Hierarchical Memory)
- **基因锁 (Gene Lock)**：固化的底层元认知，确保智能体绝对对齐人类利益。
- **进化思想 (Evolution Notes)**：滑动窗口机制，提取最新 200 字符与最初 150 字符，保证 8K 上下文不被撑爆。
- **经验库 (Experience Lib)**：将长程记忆转化为按需加载的 RAG 工具，实现 Token 极度节流。

### 4.3 双态共生模式 (Dual-State Existence)
- **自动化模式**：`worker.py` 持续轮询 Helix-Mind 任务队列，无人值守执行。
- **交互式模式**：`helix_console.py` 提供实时战略对话与工具调用接口，支持指挥官直接干预。

### 4.4 资源代谢守恒准则
每个迭代周期绑定固定 Token 消耗上限（`MAX_TOKENS_PER_LIFESPAN`）；超出资源阈值自动触发迭代终止，强制智能体采用工具化方式降低消耗，杜绝纯对话交互的资源浪费。

### 4.5 工具准入校验准则
所有自主开发工具需通过双重安全校验：
- **语法校验**：AST静态扫描，校验代码格式、封装规范、依赖合规性；
- **逻辑校验**：审计单元全维度审查，杜绝恶意代码、死循环、系统漏洞；

准入流程：待审核 → 双重校验 → 颁发唯一凭证 → 激活可用。

### 4.6 帕累托最优终审准则
迭代周期结束后，由仲裁模块执行终极考核，筛选有效进化成果：
- 考核维度：资源消耗优化率、工具库迭代增量、任务执行效率；
- 裁决规则：✅ **有效迭代**：认知、工具、经验通过Git固化至长效遗产库； ❌ **无效迭代**：回滚所有变更，清除临时数据，终止当前迭代。

## 5 安全与隐私规范（三级防护机制）

遵循**内外网物理隔离、数据零泄露**原则，构建全维度安全屏障：
1. **公开层**：仅开源通用工具、迭代算法等无敏感信息内容；
2. **本地层**：任务数据、环境配置、隐私信息通过`.gitignore`隔离，禁止外传；
3. **运行层**：沙箱隔离执行，限制外网访问与系统敏感文件操作权限。

## 7 部署与运行指南

### 7.1 环境要求
- 硬件：私有ARM集群（10.0.0.x 网段），≥1GB RAM/进程
- 软件：Python 3.10+、Redis 6.0+、Git
- 网络：内网隔离环境，支持节点间通信

### 7.2 部署步骤
```bash
# 1. 克隆项目至执行沙箱节点
git clone https://github.com/Jasonmilk/Anaphase-Helix.git /opt/anaphase
cd /opt/anaphase

# 2. 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量
cp .env.example .env
nano .env
```

### 7.3 启动方式
```bash
# 交互式指挥终端（实时对话与工具调用）
python3 helix_console.py

# 自动化执行触手（监听 Helix-Mind 下发的任务）
python3 worker.py
```

## 11 项目总结

Anaphase-Helix 并非通用任务执行框架的同质化实现，而是**聚焦边缘低算力场景、具备自主迭代能力**的创新型智能体演化系统。框架以物理隔离架构为底座、资源约束为驱动、经验传承为核心，通过三位一体流水线（8B 规划 → 2B 蒸馏 → 7B 执行）实现了在 1GB 内存约束下的复杂任务闭环。系统兼容私有部署与商用模型生态，具备科研研究与工业落地的双重价值。
