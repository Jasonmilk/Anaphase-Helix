# Anaphase-Helix: 面向私有ARM边缘集群的自主智能体迭代演化框架
**自主进化 | 资源约束适配 | 物理隔离架构 | 分层认知 | 商用模型兼容**
> 研究定位：低算力边缘环境下，基于资源最优原则的大模型智能体可持续演化系统

---

## 摘要
Anaphase-Helix 是一款专为**私有内网ARM硬件集群**设计的自主智能体演化框架，聚焦1GB RAM低算力、高延迟边缘场景，构建了**生命周期迭代-工具化落地-经验固化-安全审计-最优筛选**的全闭环演化体系。框架摒弃传统大模型纯对话交互模式，以资源代谢守恒、工具化落地、帕累托最优为核心准则，实现智能体从被动任务执行向主动进化迭代的升级。系统支持本地私有模型（Tuck网关）与商用模型服务（OneAPI生态）无缝切换，通过物理级三层隔离架构保障运行安全，是面向边缘计算场景的轻量化自主智能体研究与落地平台。

## 1 项目概述
### 1.1 研究背景
传统大模型智能体框架（如OpenClaw）普遍依赖高算力硬件与公有云服务，在ARM架构、1GB RAM等低资源边缘环境中存在算力过载、延迟过高、资源浪费等问题；同时，多数框架仅支持被动任务执行，缺乏可持续的自我迭代与经验传承能力。

### 1.2 核心目标
本框架以**边缘低算力环境适配**为基础，以**智能体自主迭代演化**为核心，实现三大技术目标：
1. **工具化落地**：将重复业务逻辑封装为标准化Python工具，降低Token资源消耗；
2. **经验固化传承**：迭代周期内提炼有效认知，形成可复用的优化补丁，实现持续能力提升；
3. **资源最优平衡**：在计算资源消耗与智能体生产力之间建立帕累托最优机制。

### 1.3 核心优势
- 物理级三层隔离架构，杜绝单点故障与代码扩散风险；
- 原生适配ARM 1GB RAM低算力边缘节点；
- 模型层完全解耦，支持私有部署与商用服务无缝切换；
- 全流程安全审计，保障智能体执行合规性；
- 可回溯的迭代演化体系，支持智能体持续进化。

## 2 物理隔离架构（三位一体分布式拓扑）
系统严格部署于 `10.0.0.x` 私有内网，采用**算力-调度-执行**物理分离架构，满足工业级安全与稳定性要求：

| 节点IP       | 模块定位       | 核心组件                | 核心职能                                                                 |
| :----------- | :------------- | :---------------------- | :----------------------------------------------------------------------- |
| **10.0.0.2** | 认知中枢       | Redis + 状态监控模块    | 管理智能体生命周期、Token消耗、运行状态，负责热数据中转与持久化存储       |
| **10.0.0.54**| 模型调度网关   | Tuck 人格路由引擎       | 多模型实例调度、人格隔离、请求路由，支持2B/4B/8B参数模型动态分配          |
| **10.0.0.206**| 安全执行沙箱  | Anaphase 内核           | 任务隔离执行、代码安全扫描、迭代成果校验、Git版本固化与回滚控制           |

> 架构特性：三层物理隔离设计，实现**认知管理、模型调度、安全执行**的解耦，兼顾演化效率与系统安全性。

## 3 智能体分工体系（三权分立协作机制）
基于Tuck网关实现多模型动态调度，构建**执行-审计-精炼**的标准化协作闭环，提升智能体任务执行与迭代效率：

### 3.1 执行单元（Worker - Qwen 4B）
- 角色定位：一线任务执行与工具开发单元
- 核心能力：任务拆解、代码实现、工具编写、业务逻辑落地
- 输出产物：Python功能脚本、任务执行日志、临时解决方案

### 3.2 审计单元（Auditor - DeepSeek R1 8B）
- 角色定位：代码安全与效率审计单元
- 核心能力：AST语法扫描、漏洞检测、性能评估、恶意行为判定
- 输出产物：安全审计报告、工具准入凭证、合规性认证

### 3.3 精炼单元（Refiner - Qwen 2B）
- 角色定位：认知经验提炼与压缩单元
- 核心能力：上下文精简、有效经验萃取、冗余数据剔除、轻量化补丁生成
- 输出产物：迭代优化补丁、固化认知规则、长效经验库

## 4 核心演化机制（三大约束准则）
框架通过**不可逆的资源与安全约束**，驱动智能体主动优化行为，实现高效自主演化：

### 4.1 资源代谢守恒准则
- 约束规则：每个迭代周期绑定固定Token消耗上限（`MAX_TOKENS_PER_LIFESPAN`）；
- 熔断机制：超出资源阈值自动触发迭代终止，强制智能体采用工具化方式降低消耗；
- 设计目的：杜绝纯对话交互的资源浪费，倒逼智能体工具化落地。

### 4.2 工具准入校验准则
所有自主开发工具需通过**双重安全校验**，方可纳入可用工具库：
1. **语法校验**：AST静态扫描，校验代码格式、封装规范、依赖合规性；
2. **逻辑校验**：审计单元全维度审查，杜绝恶意代码、死循环、系统漏洞；
- 准入流程：待审核 → 双重校验 → 颁发唯一凭证 → 激活可用。

### 4.3 帕累托最优终审准则
迭代周期结束后，由仲裁模块执行终极考核，筛选有效进化成果：
- 考核维度：资源消耗优化率、工具库迭代增量、任务执行效率；
- 裁决规则：
  ✅ **有效迭代**：认知、工具、经验通过Git固化至长效遗产库；
  ❌ **无效迭代**：回滚所有变更，清除临时数据，终止当前迭代。

## 5 安全与隐私规范（三级防护机制）
遵循**内外网物理隔离、数据零泄露**原则，构建全维度安全屏障：
1. **公开层**：仅开源通用工具、迭代算法等无敏感信息内容；
2. **本地层**：任务数据、环境配置、隐私信息通过`.gitignore`隔离，禁止外传；
3. **运行层**：沙箱隔离执行，限制外网访问与系统敏感文件操作权限。

## 7 部署与运行指南
### 7.1 环境要求
- 硬件：私有ARM集群（10.0.0.x 网段），≥1GB RAM
- 软件：Python 3.10+、Redis 6.0+、Git
- 网络：内网隔离环境，支持节点间通信

### 7.2 部署步骤
```bash
# 1. 克隆项目至执行沙箱节点
git clone https://github.com/Jasonmilk/Anaphase-Helix.git /opt/anaphase
cd /opt/anaphase

# 2. 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量
cp .env.example .env
nano .env
```

### 7.3 启动方式
```bash
# 单周期迭代（测试/调试）
python3 arbiter_loop.py

# 连续演化（生产环境）
python3 run_evolution.py
```
